# PowerCapslock v0.1.2.preview

## Highlights

- Fixed the `CapsLock + A` regression where `A` could leak to the system before speech input started.
- Moved the speech start/stop work off the low-level keyboard hook path so the hook returns immediately after interception.
- Added a built-in `--test-capslock-a` self-test path and a Python validation script under `test/`.
- Standardized runtime logs under `%USERPROFILE%\.PowerCapslock\logs\`.
- Added a release packaging flow that includes the required runtime DLLs and release-friendly install/uninstall scripts.

## What Changed

### Hook handling

- `CapsLock + A` is now intercepted first and speech startup is queued back to the app thread asynchronously.
- Speech stop is also handled asynchronously to keep the hook callback short and predictable.
- The message loop now explicitly handles hook-owned messages and audio capture timer messages.

### Validation

- Added a built-in hook self-test entrypoint:
  - `powercapslock.exe --test-capslock-a <json-report>`
- Added `test/capslock_a_self_test.py` to:
  - run the built-in self-test
  - read the latest log from `%USERPROFILE%\.PowerCapslock\logs\`
  - export JSON, stdout, log excerpt, and a Markdown report under `test/output`

### Packaging

- Preview zip now includes:
  - `powercapslock.exe`
  - `onnxruntime.dll`
  - `onnxruntime_providers_shared.dll`
  - `sherpa-onnx-c-api.dll`
  - `sherpa-onnx-cxx-api.dll`
  - release `install.bat` and `uninstall.bat`
  - default config and icon resources
- Added `scripts/package_release.ps1` to generate the preview archive into `.release/`.
